# セキュリティ

このプロジェクトでは、ソフトウェア製品とサービスのセキュリティを真剣に取り組んでいます。

## セキュリティ問題の報告

このリポジトリでセキュリティ脆弱性を発見したと思われる場合は、調整された開示を通じて報告してください。

**パブリックなGitHubのissue、ディスカッション、またはプルリクエストを通じてセキュリティ脆弱性を報告しないでください。**

代わりに、以下の方法でご連絡ください：

- GitHubのissueでセキュリティラベルを付けて報告
- または、プロジェクトメンテナーに直接連絡

問題をより良く理解し解決するために、以下の情報をできるだけ多く含めてください：

  * 問題の種類（例：バッファオーバーフロー、SQLインジェクション、クロスサイトスクリプティング）
  * 問題の現れに関連するソースファイルの完全パス
  * 影響を受けるソースコードの場所（タグ/ブランチ/コミットまたは直接URL）
  * 問題を再現するために必要な特別な設定
  * 問題を再現するためのステップバイステップの手順
  * 概念実証またはエクスプロイトコード（可能であれば）
  * 攻撃者が問題をどのように悪用する可能性があるかを含む、問題の影響

この情報は、レポートをより迅速にトリアージするのに役立ちます。

## ポリシー

セキュリティ問題の責任ある開示にご協力いただき、ありがとうございます。報告された問題は適切に調査し、必要に応じて修正いたします。

# Security

This project takes the security of our software products and services seriously.

## Reporting Security Issues

If you believe you have found a security vulnerability in this repository, please report it to us through coordinated disclosure.

**Please do not report security vulnerabilities through public GitHub issues, discussions, or pull requests.**

Instead, please contact us using one of the following methods:

- Report through GitHub issues with a security label
- Or contact project maintainers directly

Please include as much of the information listed below as you can to help us better understand and resolve the issue:

  * The type of issue (e.g., buffer overflow, SQL injection, or cross-site scripting)
  * Full paths of source file(s) related to the manifestation of the issue
  * The location of the affected source code (tag/branch/commit or direct URL)
  * Any special configuration required to reproduce the issue
  * Step-by-step instructions to reproduce the issue
  * Proof-of-concept or exploit code (if possible)
  * Impact of the issue, including how an attacker might exploit the issue

This information will help us triage your report more quickly.

## Policy

Thank you for your cooperation in responsibly disclosing security issues. Reported issues will be properly investigated and remediated as necessary.
