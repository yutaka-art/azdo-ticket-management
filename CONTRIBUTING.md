## プルリクエストの提出

1. リポジトリを[フォーク][fork]してクローンしてください
1. 依存関係を復元してください: `dotnet restore`
1. お使いのマシンでアプリケーションがビルドできることを確認してください: `dotnet build`
1. お使いのマシンでアプリケーションが正常に動作することを確認してください
1. 新しいブランチを作成してください: `git checkout -b my-branch-name`
1. 変更を加え、動作確認を行い、ビルドが引き続き通ることを確認してください
1. フォークにプッシュして[プルリクエストを提出][pr]してください
1. 自分を褒めて、プルリクエストがレビューされマージされるのを待ちましょう

プルリクエストが受け入れられる可能性を高めるためにできることがいくつかあります：

- 動作確認を行ってください。
- 変更をできるだけ集中させてください。相互に依存しない複数の変更を行いたい場合は、別々のプルリクエストとして提出することを検討してください。
- [良いコミットメッセージ](http://tbaggery.com/2008/04/19/a-note-about-git-commit-messages.html)を書いてください。

## Submitting a pull request

1. [Fork][fork] and clone the repository
1. Restore the dependencies: `dotnet restore`
1. Make sure the application builds on your machine: `dotnet build`
1. Make sure the application works properly on your machine
1. Create a new branch: `git checkout -b my-branch-name`
1. Make your change, verify functionality, and make sure the build still passes
1. Push to your fork and [submit a pull request][pr]
1. Pat yourself on the back and wait for your pull request to be reviewed and merged.

Here are a few things you can do that will increase the likelihood of your pull request being accepted:

- Verify functionality.
- Keep your change as focused as possible. If there are multiple changes you would like to make that are not dependent upon each other, consider submitting them as separate pull requests.
- Write a [good commit message](http://tbaggery.com/2008/04/19/a-note-about-git-commit-messages.html).
